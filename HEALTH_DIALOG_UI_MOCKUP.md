## Interactive Elements

1. **Auto-refresh Toggle**: Enable/disable automatic refresh of health and session data
2. **Refresh Interval**: Configure refresh interval (5-300 seconds)
3. **Refresh Button**: Manual refresh of all data
4. **Export Button**: Export logs (functionality to be implemented)
5. **View Details Button**: Expand session information panel
6. **Terminate Button**: Terminate individual session (with loading state)
7. **Terminate Other Sessions**: Bulk terminate all non-current sessions (with confirmation)

## Responsive Behavior

- **Desktop (md+)**: Two-column layout (Health + Sessions side by side, Logs full width below)
- **Tablet (sm)**: Stacked layout (Health, then Sessions, then Logs)
- **Mobile (xs)**: Single column with compressed tables

## Color Scheme

- **Success (Healthy)**: Green
- **Warning (Degraded)**: Orange/Yellow
- **Error (Unhealthy)**: Red
- **Info**: Blue
- **Current Session**: Success Green chip
- **Roles**: Secondary colored chips
- **Buttons**: 
  - Primary for actions
  - Warning for terminate operations
  - Default for view operations

## Accessibility Features

- All interactive elements have proper `aria-label` attributes
- Keyboard navigation supported (Tab, Enter, Escape)
- Screen reader friendly with descriptive labels
- Color coding supplemented with icons and text
- Focus indicators on all interactive elements

## Loading States

- Circular progress indicator when fetching initial data
- Disabled buttons with visual feedback during operations
- Snackbar notifications for operation completion
- Graceful degradation if session data unavailable

## Error States

- Warning alert if sessions cannot be loaded
- Info alert if no active sessions
- Error snackbar for failed termination attempts
- Console logging for debugging (doesn't affect UX)
